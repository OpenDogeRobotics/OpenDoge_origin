# OpenDoge 开发计划

## 项目简介

OpenDoge 是一款开源的小型四足机器人项目，涵盖从机械结构设计、URDF 仿真描述、嵌入式固件控制（ROS2）到强化学习训练与实机部署的全栈开发。目标是打造一套低成本、可复现、端到端的四足机器人开发方案，降低四足机器人研发的门槛。

- [x] 预训练模型权重（galileo、g1、h1、h1_2、zsl1）

## 二、技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| **机械设计** | SolidWorks | 零件设计、装配体导出 STEP |
| **仿真模型** | URDF / Xacro | 运动学与动力学描述 |
| **物理仿真** | Isaac Gym + Mujoco + Gazebo | Isaac Gym 用于 RL 训练，Mujoco 用于 Sim2Sim 验证，Gazebo 用于 ROS 仿真 |
| **机器人中间件** | ROS2 Humble | 固件层通信与控制 |
| **硬件控制** | ros2_control + CAN | 实时电机控制总线 |
| **传感器** | DM-IMU | 姿态传感器 |
| **深度学习框架** | PyTorch + TensorFlow | PyTorch 训练 RL 模型，TensorFlow 用于 TensorBoard |
| **RL 算法库** | RSL-RL / HIMLoco | PPO 算法，ACTOR-CRITIC 架构 |
| **模型导出** | ONNX Runtime | PyTorch → ONNX 转换与推理 |
| **通信协议** | LCM | Sim2Real 部署时上位机与机器人通信 |
| **监控面板** | Flask Web Dashboard | 训练监控与策略部署 |
| **编程语言** | Python（训练/部署）、C++（电机驱动/底层） | |
| **操作系统** | Ubuntu 20.04（训练）、Windows/Ubuntu（固件部署） | |

### 关键依赖

- CUDA 12.1、PyTorch GPU、Isaac Gym Preview 4
- ROS2 Humble + ros2_control + controller_manager
- Mujoco 3.2.3、ONNX Runtime、LCM

## 三、后续开发计划

### Phase 1：OpenDoge 本体训练闭环（优先级：高）

- [ ] **OpenDoge 专属训练配置**：基于现有 URDF 创建 OpenDoge 专属的 Isaac Gym 训练任务配置，不再依赖 galileo/go2 等外部机器人配置
- [ ] **域随机化调优**：针对 3D 打印结构件的制造公差，增加动力学参数域随机化（质量、惯量、摩擦力、电机力矩系数）
- [ ] **Sim2Sim 验证流程**：编写 OpenDoge 专属的 Mujoco 验证脚本，验证策略在 Mujoco 中的迁移效果
- [ ] **开环行走策略**：训练出第一个能够在仿真中稳定行走的 OpenDoge 策略
- [ ] **策略网络导出与量纲对齐**：导出 ONNX 模型，完成仿真策略与真实电机量纲（N·m）的对齐

### Phase 2：实机部署与调试（优先级：高）

- [ ] **电机标定与正方向确认**：完成 12 个电机的 ID 映射、正方向测试、力矩常数标定
- [ ] **IMU 与机器人本体标定**：IMU 安装误差补偿、机身坐标系对齐
- [ ] **CAN 总线稳定性测试**：长时间运行下的通信丢包与延迟测试
- [ ] **RK3588 RL 策略实机推理部署**：通过 `opendoge_rl_node` 在 RK3588 上完成策略推理闭环；电机 `joint_state` 与 `/imu` 作为模型输入，策略输出 POS，节点以 50 Hz 推理、200 Hz 发布 `/joint_target`，底层电机控制保持 500 Hz
- [ ] **POS 目标通信桥接**：将 `/joint_target` POS 目标接入 LCM 或 DDS 通信桥，再由底层驱动转换为 EL05/RobStride 控制帧；桥接层需保留话题频率统计、超时保护和关节顺序校验
- [ ] **手柄方向控制结构**：实现 `/joy` 手柄输入到速度指令和状态机的映射，支持 passive、standby、running、fault/estop，并对 x/y/yaw 指令做死区、限幅和 deadman 保护
- [ ] **站立与行走实机验证**：从站立策略开始，逐步过渡到原地踏步、低速行走

#### Phase 2 当前实现记录（2026-05-22）

- [x] 在 `OpenDoge_firmware/src/opendoge_rl_node` 中实现 RK3588 推理桥接节点骨架。
- [x] 节点输入：`/joint_state` (`sensor_msgs/JointState`)、`/imu` (`sensor_msgs/Imu`)、`/joy` (`sensor_msgs/Joy`)、`/cmd_vel` (`geometry_msgs/Twist`)。
- [x] 节点输出：`/joint_target` (`sensor_msgs/JointState`)，按 12 关节 POS 目标发布；同时发布 `/rl_observation` 与 `/rl_action` 便于调试。
- [x] 控制节拍：参数化 `control_rate_hz=500`、`publish_rate_hz=200`、`inference_rate_hz=50`，当前节点负责 50 Hz 策略更新和 200 Hz 目标保持发布。
- [x] 手柄结构：通过 deadman、standby、running、passive、estop 按钮和 x/y/yaw 轴映射控制机器人模式与方向命令。
- [x] 参考 HighTorque `sim2real-inference_code_ros2` 的多回调组结构、observation 构造、动作发布、joy 状态切换和关节顺序映射方式。
- [ ] RKNN/ONNX 真实模型后端：当前保留 `policy_backend=rknn` 接入点，需在 RK3588 安装 Rockchip `rknnrt` 后替换为真实 NPU 推理实现。
- [ ] LCM/DDS 目标桥：当前先以 ROS2 `/joint_target` 为标准出口，后续由电机桥接包负责转 LCM/DDS 或直接转 EL05 CAN 帧。

### Phase 3：运动能力提升（优先级：中）

- [ ] **步态规划**：实现多种步态（trot、walk、bound、pace）与步态切换
- [ ] **速度追踪**：准确跟踪目标线速度与角速度指令
- [ ] **地形适应**：训练上下坡、楼梯、不平整地面行走能力
- [ ] **抗扰恢复**：外力扰动下的平衡恢复能力
- [ ] **摔倒检测与自起立**：检测摔倒状态并执行起立动作

### Phase 4：感知与自主性（优先级：中）

- [ ] **机身状态估计**：基于 IMU + 关节编码器的本体位姿估计（扩展卡尔曼滤波 / 因子图）
- [ ] **深度相机集成**：搭载 Intel RealSense 或 OAK-D 深度相机
- [ ] **视觉-运动融合策略**：将深度/高程图作为策略网络输入，实现视觉引导的地形适应
- [ ] **SLAM 导航**：集成 RTAB-Map 或 SLAM Toolbox，实现自主导航

### Phase 5：工具链与社区建设（优先级：低）

- [ ] **一键训练脚本**：从 URDF 到训练模型的自动化 Pipeline
- [ ] **硬件 BOM 清单**：完整元器件采购清单与成本估算
- [ ] **组装教程**：图文+视频组装指南
- [ ] **仿真 Docker 镜像**：打包 Isaac Gym 训练环境为 Docker 镜像，降低环境配置门槛
- [ ] **Web Dashboard 完善**：训练监控 + 模型管理 + 一键部署面板
- [ ] **多平台固件支持**：探索 micro-ROS 或裸机 FreeRTOS 方案，摆脱上位机依赖
- [ ] **开源社区运营**：完善文档、Issue 模板、PR 流程、Discord 社区

## 四、当前风险与挑战

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 3D 打印零件一致性差 | 动力学参数偏离仿真，Sim2Real Gap 大 | 域随机化训练 + 电机力矩裕量设计 |
| CAN 总线通信延迟 | 控制频率不稳定 | 优化总线负载，提高通信波特率 |
| Isaac Gym 闭源 | Isaac Gym 停止维护后无法复现训练 | 推进 Mujoco MJX 迁移，探索 Isaac Lab 替代方案 |
| 电机驱动器一致性 | 各关节力矩响应差异大 | 逐关节标定 + 系统辨识 |
| 实机跌倒损坏 | 电机/结构损坏 | 急停逻辑 + 软限位 + 保护笼/绳索悬挂初始测试 |
